# Система электронного документооборота (СЭД)

Программный модуль для автоматизации создания, согласования, подписания и хранения документов предприятия.

## Стек технологий

**Backend:**
- ASP.NET Core (.NET 8.0)
- Entity Framework Core
- JWT-аутентификация
- BCrypt для хеширования паролей
- Swagger для документации API

**Frontend:**
- React 18
- TypeScript
- React Router
- Axios
- Vite

**База данных:**
- PostgreSQL

## Функциональность

- Регистрация и авторизация через JWT-токены
- Четыре роли: Администратор, Сотрудник, Согласующий, Руководитель
- Создание документов четырёх типов: приказы, договоры, служебные записки, письма
- Последовательный и параллельный маршруты согласования
- Подписание, отклонение и запрос доработки документов
- Контроль версий файлов
- Поиск и фильтрация документов по статусу, типу и названию
- Личный кабинет с задачами на подпись
- Уведомления о новых документах и смене статуса
- Разграничение прав доступа на основе ролей

## Установка и запуск

### 1. Установите программы

- [.NET 8.0 SDK](https://dotnet.microsoft.com/download)
- [Node.js 18+](https://nodejs.org)
- [PostgreSQL](https://www.postgresql.org/download)

### 2. Создайте базу данных

Откройте pgAdmin. Создайте базу данных с именем `sed_db`.

### 3. Настройте строку подключения

В файле `SED.Api/appsettings.json` замените `твой_пароль` на пароль пользователя postgres:

```json
"DefaultConnection": "Host=localhost;Database=sed_db;Username=postgres;Password=ваш_пароль"
